Teardown

Teardown is a voxel-based heist and sandbox game with raytraced graphics and destruction physics.

Level format

Modded levels are in XML format. The root element of a level is a <scene> element. This is also the original format of the base game levels, but the source files of the base game levels are not included in the release builds' files. They are baked into a proprietary binary format referred to by the community as "TDBIN". Binary levels are stored in data/bin/, have .bin extensions, and are compressed with DEFLATE and can be decompressed/recompressed with zlib-flate—except for quicksaves, which are uncompressed and located at %LOCALAPPDATA%/Teardown/quicksave.bin.

Scripting

Teardown embeds Lua 5.1.4. It does not "open" the package, I/O, OS, or debug libraries, which means no require, io.open, os.execute, or debug.getinfo. You still have dofile and loadfile though (although you're not supposed to and they don't respect MOD/ or LEVEL/ notation). Loading of bytecode has been disabled.

I intend to document mod support thoroughly, but for now, most of the information you could want to know is on the game's website.

Versions

Please ensure you are familiar with Steam's technical vocabulary.
The "Ver." column refers to the value of the game.version registry node (game registry, not Windows registry!).
The "Patch" column refers to the value of the game.version.patch registry node. 0.5.5 (6205729) was the earliest build to have support for this, and 0.6.2 (patch 1) (6324468) was the first build to make use of it.
Rows without a "buildid" value are non-Steam builds.
Rows without a "manifests" value are builds I don't have access to because they were never on a public branch.
Manifest IDs are always for depot ID 1167634. Another Teardown depot is known to exist (1167631), but it has no known builds.
Branches are separated by commas and go in order of first branch the build appeared on to last.
The "CRC-32" column is the CRC-32 checksum of the game's executable, i.e. teardown.exe (except for perftests which are teardown-perftest.exe), in hexadecimal (hence the 0x at the beginning).
Rows without a "CRC-32" value are builds I have not checked, either because I don't have that build, or because I just haven't gotten around to downloading it yet.
